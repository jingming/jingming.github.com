API Definitions
With the growing number of teams and products at Twilio, it became unsustainable to have the team write code for each API that Twilio introduced. As a result, we extended the Helper Library Generation platform that was built into the API Proxy so that teams would be able to define the API Proxy as well as the Helper Libraries. Technologies Used: Python
Helper Library Generation
Twilio provides Helper Libraries in 6 languages (C#, Java, node.js, PHP, Python, Ruby) to simplify building on top of the API. Since they are mostly wrappers around the API, the team started building a tool to auto-generate these libraries when a change was made in the API. To start, I was in charge of building the Java and node.js libraries. After some organizational changes, I took over leadership of this project and drove it to completion with all 6 libraries being generated through the tool. This brought down the time to update all the libraries from roughly 2 weeks of engineering time (for one engineer) to 30 seconds on top of significantly reducing the number of bugs caused by removing humans from the flow.
Technologies Used: Python, node.js, Java, C#, Ruby, PHP
